# Project Rules & Preferences

This file documents the coding style and guidelines defined by the project owner. All agents and developers must strictly follow these guidelines:

## Styling & Layout Rules
* **No Tailwind CSS**: Do not use Tailwind CSS in this project. The owner is not interested in Tailwind CSS for this or any of their other projects.
* **Vanilla CSS**: Always write custom, clean Vanilla CSS for styling.
* **Mobile-First Approach**: Design responsive layouts focusing on mobile viewports first, using min-width media queries to scale up to desktop screens.
